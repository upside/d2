# Asset Specs — чтобы это реально версталось (v0.1)

## 1) Форматы

- UI элементы: PNG (прозрачность) или WebP (если нужен размер).
- Большие фоны/мокапы: WebP/JPG (без прозрачности).
- Иконки: PNG с прозрачностью.


## 1.5) Прозрачность и alpha (обязательный контракт)

**Для всех ассетов типа `icon/*` (предметы, руны, камни, зелья, персонажи-иконки):**

- **Фон полностью прозрачный (RGBA alpha).** Никакого чёрного/серого/цветного фона, градиентов, виньетки, “карточек”, “плашек”, окружения.
- Допустим мягкий `drop shadow`/AO только **в альфа-канале** (тень “в воздухе”), без “пола”/поверхности.
- Держи “safe padding” по краям: **8–12 px** пустого пространства (UI не должен резать контур).
- Если ассет по ТЗ требует рамку/подложку (например, слот инвентаря) — это **не icon**, а `ui/*` и делается отдельно.

**Проверка перед коммитом:** открыть PNG на шахматной подложке (или в редакторе) и убедиться, что за объектом прозрачность, а не заливка.

## 2) Сетки и размеры (рекомендуемые)

### Иконки предметов
- 128x128 (основной)
- 256x256 (крупные/герой)
- Внутренний safe‑margin: 10–14%

### Инвентарь
- Slot: 64x64 (основа)
- Large slot: 80x80 (для экипировки)
- Socket: 24x24 (основа), 28x28 (крупнее)

### Кнопки
- Высота: 32 / 40 / 48
- Минимальный радиус/скос: визуально одинаковый

### Tooltip
- Ширина: 360–440 (max 480)
- Паддинги: 16–20

## 3) 9‑slice (обязательно для рамок)

Для panel/frame ассетов:
- Углы: 24px (или 32px для больших)
- Борты: повторяемый паттерн, без “уникальных” трещин, иначе будет видно тайлинг.
- Центр: максимально спокойный/чистый (под текст).

## 4) Именование файлов (строго)

Пример:
- ui/panel/panel_inventory_9slice.png
- ui/button/btn_primary_md_default.png
- ui/button/btn_primary_md_hover.png
- ui/slot/slot_64_empty.png
- ui/slot/slot_64_hover.png
- ui/socket/socket_24_empty.png
- ui/tooltip/tooltip_frame_9slice.png
- icon/item/armor_chest_128.png
- icon/rune/rune_01_64.png

Правило: lowercase, snake_case, без пробелов.

## 5) Визуальные ограничения (для консистентности)

- Текстуры: не сильнее 20–30% субъективной “силы” от текста/иконок.
- Glow: только локальный и мягкий, не вокруг всего объекта.
- Тени: 2 уровня (soft / deep). Не делать 5 разных “стилей тени”.

## 6) UI Tokens (для CSS / темизации)

> Это не “дизайн система”, а якоря.

- Spacing: 4 / 8 / 12 / 16 / 24 / 32
- Radius/Skew: 4 / 8 (или “скос 6px”)
- Shadow: soft / deep
- Border: thin (1px), medium (2px)
- Typography: base 14–16, title 18–22

## 7) Готовность ассета (чек‑лист)

- [ ] **Иконки (`icon/*`): фон 100% прозрачный (alpha).** Никаких фонов/градиентов/виньетки/подложек.
- [ ] Центровка и одинаковые отступы
- [ ] Пивот/силуэт читаемы на 64px
- [ ] Нет текста/логотипов/водяных знаков
- [ ] Стили (материалы/свет) совпадают с Style Bible
